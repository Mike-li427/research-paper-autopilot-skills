# Quality Gates

Use these gates before advancing stages or claiming readiness.

## Gate Results

- `PASS`: evidence is adequate for the requested output mode.
- `WARN_DEFERRED`: acceptable only in a provisional draft with explicit rationale or follow-up.
- `HARD_BLOCK`: cannot enter a submission-ready package.
- `OUT_OF_SCOPE`: allowed only when the affected claim/artifact is removed.

## Literature Gate

Block or refine when:

- Sources are generated by an LLM rather than retrieved from real databases, proceedings, user PDFs, or official project pages.
- Key records lack verifiable metadata when DOI/arXiv/URL should exist.
- Records lack provenance fields such as source database, retrieval date, metadata source, or verification status.
- The corpus is too small or padded with off-topic prestigious papers.
- Claims about "first", "state of the art", "novel", or "no prior work" are not backed by search strategy and screened records.

Repair route: `LITERATURE_DISCOVERY` or `LITERATURE_SCREEN`.

## Method And Experiment Gate

Block or refine when:

- No dataset, corpus, proof object, empirical design, or evidence source can support the claim.
- Experiment or analysis code was written without applying `references/experiment-code-standard.md`.
- Baselines are weak, untuned, missing, or incomparable.
- Ablations do not cover named components.
- Metrics do not match the research question.
- Pilot run, runtime estimate, seeds, configs, logs, or convergence criteria are missing.
- NaN/Inf/runtime failures are hidden instead of diagnosed and repaired.

Repair route: `METHOD_OR_EXPERIMENT_DESIGN` or `EXECUTION_OR_ANALYSIS`.

## Code Quality Gate

Block or refine when:

- Code hardcodes paper-facing metrics, p-values, curves, tables, or "best" results.
- Mock data, random curves, or placeholder metrics are presented as experiment evidence.
- Scripts cannot be run from a documented command or do not write results/logs.
- Config, seed, dataset path, output path, and experiment mode are not explicit.
- File names, function names, variable names, or metric names are generic boilerplate rather than project-specific.
- Comments or docstrings explain obvious operations instead of research choices, edge cases, or non-obvious assumptions.
- One large script mixes data loading, method, baselines, metrics, plotting, and paper export when a smaller project-specific split is feasible.
- The code looks polished but empty: broad helpers, unused abstractions, vague names, no saved artifacts, and no connection to the paper's claim-evidence map.

Repair route: `EXECUTION_OR_ANALYSIS`.

## Result Artifact Gate

Block or refine when:

- Result files do not include raw per-condition records, aggregate summaries, config snapshot/hash, command line, seed, log path, and environment or code metadata.
- A summary file exists without raw run records or archived computation.
- Malformed JSON/CSV, non-finite metrics, failed/crashed/error runs, or hidden dropped data appear in result artifacts.
- Statistical tests are claimed without archived test name, statistic, p-value or interval, sample size, command/hash, and source result file.
- Numeric claims are matched only by global number coincidence rather than claim-evidence entries or table/figure mappings.

Repair route: `EXECUTION_OR_ANALYSIS` or `RESULT_INTERPRETATION`.

## Multi-Agent Code Review Gate

Block or refine when:

- Experiment code, analysis code, result files, or experiment/statistical manuscript claims exist but `audit/code_review_ledger.md` or `.json` is missing.
- `audit/code_review_packet.md` is missing, stale, truncated without inventory, or not bound to the ledger by `packet_sha256`.
- The ledger does not contain all six mandatory roles from `multi-agent-code-review.md`.
- Any mandatory reviewer role is `PENDING`, `BLOCK`, missing a status, missing a reviewer run/session id, missing evidence, or missing findings.
- Any `WARN` lacks an explicit accepted rationale or follow-up test plan.
- Any `BLOCK` lacks required fixes.
- Multiple ledgers conflict, duplicate roles occur, or a previous `BLOCK` is hidden by a later `PASS`.
- The project used a single-agent self-review as a substitute for the required independent role reviews.
- Result interpretation, tables, figures, or manuscript claims were written before the multi-agent code review gate passed.

Repair route: `MULTI_AGENT_CODE_REVIEW`, then `EXECUTION_OR_ANALYSIS` if code/results changed.

## Evidence Gate

Block or refine when:

- Numeric claims in the paper do not match result files, tables, or claim-evidence mappings.
- Statistical tests are claimed but no archived computation supports them.
- Figures/tables are not reproducible from data or result artifacts.
- Limitations contradict the results or omit known failure cases.
- A claim has no support from results, code, data, literature, official docs, or explicit derivation.
- The claim-evidence map is empty, marks unsupported claims as supported, or references missing evidence paths.

Repair route: the exact evidence-producing stage.

## Writing Gate

Block or refine when:

- The story drifts away from the stated research question.
- The paper overclaims beyond the evidence.
- Environment setup, tool failures, or agent process are framed as contributions.
- The contribution list has more than two core innovations without a clear hierarchy.
- Abstract, introduction, method, experiments, and conclusion disagree on the main claim.
- A hard blocker is deferred while the output is labeled submission-ready.

Repair route: `PAPER_ARCHITECTURE`, `DRAFTING`, or evidence-producing stages.

## Review Gate

Block or refine when:

- Simulated reviewers identify fatal novelty, evidence, identification, reproducibility, ethics, or formatting issues.
- Reviewer objections are acknowledged but not translated into edits, experiments, citations, or limitations.
- A rebuttal promises changes absent from the manuscript or artifacts.

Repair route: `REVISION` plus the exact upstream repair stage.

## Submission Gate

Block or refine when:

- Target venue rules, anonymity, page limits, figure formats, references, supplementary files, disclosures, data/code availability, or repository requirements are unknown or unmet.
- A compiled PDF, source package, figures, bibliography, and reproducibility/readme files are not present when required.
- Restricted data, human subjects, medical, privacy, or licensing issues lack a statement or handling plan.
- Submitted or accepted status is claimed without an explicit user-provided or external evidence artifact.

Repair route: `SUBMISSION_PACKAGE`, `INTEGRITY_AUDIT`, or target-track specialist repair.
