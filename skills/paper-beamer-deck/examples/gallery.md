# Examples Gallery

This gallery intentionally uses synthetic or generated examples only.
Do not add copied paper figures, paper text, or screenshots from copyrighted
papers here unless their license explicitly permits redistribution.

## Figure Extraction Preview

The figure extraction pipeline can be demonstrated with a tiny synthetic PDF
generated by the test suite. It exercises the same output structure as a real
paper:

```text
build/figures/demo/
  figures.json
  figures/
  pages/
  previews/index.html
  figure_overrides.example.json
```

What to look for in the preview:

- crop image for each `status: ok` record;
- red bbox overlay on the rendered source page;
- label, page, type, method, bbox, caption, and reason fields;
- explicit `needs_manual_review` records when no reliable bbox exists.

## Deck Quality Report

For a generated deck, the quality report summarizes the parts a presenter cares
about before using the slides:

```bash
python3 skills/paper-beamer-deck/scripts/generate_quality_report.py \
  slides/demo --figures-json build/figures/demo/figures.json
```

The report records compilation/log status, visual-QA coverage, figure extraction
status, and open risks. It is a review aid, not a replacement for reading the
paper or inspecting the final PDF.
